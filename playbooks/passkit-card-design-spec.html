PassKit Field Mapping

Front Side Fields (Primary Display)

primaryFields: - key: "logo" label: "" value: "HIRECAR" textAlignment: "left" secondaryFields: - key: "memberName" label: "" value: "{{firstName}} {{lastName}}" textAlignment: "left" auxiliaryFields: - key: "memberId" label: "" value: "{{memberID}}" textAlignment: "left"

Back Side Fields (Details & QR)

backFields: - key: "tier" label: "Membership Tier" value: "{{tierName}}" - key: "joinDate" label: "Member Since" value: "{{joinDate}}" - key: "benefits" label: "Benefits" value: "{{benefitsList}}" - key: "contact" label: "Support" value: "support@hirecar.com"

Barcode / QR Code

barcodes: - format: "QR" message: "{{qrData}}" messageEncoding: "iso-8859-1" altText: "Member ID: {{memberID}}"

Dynamic Fields

The following variables should be replaced with actual member data when generating passes:

  • {{firstName}} - Member's first name
  • {{lastName}} - Member's last name
  • {{memberID}} - Unique member identifier (HC-2026-XXXXX)
  • {{tierName}} - STANDARD, OPERATOR, FIRST CLASS, or ELITE
  • {{joinDate}} - Date member joined (MM/DD/YYYY)
  • {{qrData}} - QR code content (URL or member ID)
  • {{benefitsList}} - Comma-separated benefits for tier

Implementation Guide

PassKit Dashboard Configuration

  • Team ID: Required from Apple Developer Account
  • Pass Type ID: Set consistent identifier (e.g., pass.com.hirecar.membership)
  • Certificate: Upload WWDR certificate and Pass Signing Certificate
  • Expiration: Set renewal cadence (annual recommended)
  • Logos: Provide @1x (163×61px) and @2x (326×122px) logo files

JSON Structure Example (pkpass)

{ "formatVersion": 1, "passTypeIdentifier": "pass.com.hirecar.membership", "serialNumber": "HC-2026-00001", "teamIdentifier": "YOUR_TEAM_ID", "organizationName": "HIRECAR", "description": "HIRECAR Membership Card", "foregroundColor": "rgb(17, 24, 32)", "backgroundColor": "rgb(184, 184, 184)", "labelColor": "rgb(255, 255, 255)", "logoText": "HIRECAR", "barcode": { "format": "PKBarcodeFormatQR", "message": "https://hirecar.com/member/HC-2026-00001", "messageEncoding": "iso-8859-1" } }

Tier-Specific Badge Implementation

  • Standard: Silver appearance, suitable for basic members
  • Operator: Gold/premium appearance, highlighted benefits
  • First Class: Black matte with gold accents, luxury feel
  • Elite: Animated holographic effect, premium positioning
  • Validation: Backend system validates tier before pass generation

Distribution Methods

  • Email: Send pkpass file as attachment or via secure link
  • Web: Host pass files on secure server with HTTPS
  • Mobile App: Generate dynamically within native app
  • SMS: Send passbook link via short code
  • QR Registration: Members can scan QR to add to wallet

Maintenance & Updates

  • Expiration: Passes should expire annually; prompt renewal
  • Updates: Use push notifications to update pass when tier changes
  • Revocation: Implement system to revoke compromised passes
  • Analytics: Track pass views, taps, and unlock events
  • Testing: Test on iOS Wallet and Android Pay before deployment

Quality Assurance Checklist

Visual Testing

  • All four tiers display correctly on iOS Wallet
  • All four tiers display correctly on Android Google Pay
  • Colors are accurate and meet WCAG contrast requirements
  • Text is legible at small sizes (11px minimum)
  • QR codes are scannable and high contrast
  • Gradient effects render smoothly across devices
  • Elite tier animation performs smoothly without lag

Functional Testing

  • QR code scans and directs to correct member profile
  • Member data populates correctly in all fields
  • Expiration date functionality works as expected
  • Pass updates push successfully when tier changes
  • Revocation process works without errors
  • Member can add/remove pass from wallet
  • Analytics tracking captures all events

Compliance Testing

  • All certificates are valid and current
  • Pass Type ID matches provisioning profile
  • PII is encrypted and transmitted securely
  • GDPR compliance verified for EU members
  • Signature and manifest files are valid
  • No deprecated APIs or features used
